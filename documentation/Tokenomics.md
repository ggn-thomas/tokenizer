L'authorité du token serait 42 Centrale.

Quand un étudiant effectue une correction alors automatiquement un token lui est transféré, la personne corrigé elle se fait débité dès lors qu'elle prend la correction.

Chaque nouvelle étudiant recoit un aidrop de 5 token a son arrivé.

L'idée est qu'il y est une supply fixe avec comme répartition:

50% dédiés pour les étudiants
30% liquidité dédiés pour la liquidité
20% vesting tiers

Une partie de la pool serait directement destiné pour les étudiants, plusieurs cas d'usage:

- Pour se faire corriger
- Récompense pour avoir contribuer à l'école (aide maintenance etc)
- Pour acheté des privilèges, bounties via une marketplace

Une autre partie de la pool servirait de liquidité dont 42 aurait le controle.

Une autre partie serait destiné pour le vesting, des tiers pourrait acheté le token pour contribuer au développement de l'école.

Bien sur l'étudiant n'aurait jamais a dépensé un seul centime de sa poche, tous serait automisé, l'authorité (L'école) paierait les frais de transactions.

Un élève aurait le droit de repartir avec 5 token maxium. La plue value il peut choisir de la remettre dans la pool de liquidité ou de faire un don (max 1 token par personne). Cette gestion serait gérer par un program on-chain ou via 42 Centrale
